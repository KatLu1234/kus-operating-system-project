

# [계획서] xv6 커널 내 PSI 메커니즘 구현 및 OOM 감지

## 1. 개요
본 계획은 교육용 OS인 **xv6**의 커널 공간(Kernel Space) 내에서 프로세스의 메모리 대기 시간(Stall Time)을 직접 측정하고, 이를 **지수 평균(Exponential Averaging)** 방식으로 계산하여 시스템의 메모리 압박 지표를 생성하는 것을 목표로 합니다.

## 2. 주요 단계 및 구현 로직

### Phase 1: PCB 확장 및 상태 추적 (`kernel/proc.h`)
xv6의 프로세스 제어 블록(PCB)인 `struct proc`을 수정하여 대기 시간을 기록할 변수를 추가합니다.
*   **변수 추가:** `mem_stall_ticks`(누적 대기 틱 수), `last_stall_start`(대기 시작 시점) 필드 추가.
*   **상태 정의:** 프로세스가 메모리 할당(`kalloc`) 요청 시 가용 페이지가 없어 `SLEEPING` 상태로 전환되는 시점을 '메모리 스톨(Memory Stall)' 상태로 정의합니다.

### Phase 2: 스톨 시간 측정 (`kernel/kalloc.c` & `kernel/proc.c`)
메모리 자원 부족으로 인한 멈춤 현상을 감지하고 기록합니다.
*   **kalloc 수정:** `kalloc()` 함수가 실패하여 프로세스가 대기 상태에 들어갈 때 `last_stall_start`에 현재 틱(tick)을 기록합니다.
*   **상태 복구:** 프로세스가 다시 메모리를 할당받아 `RUNNABLE` 상태가 될 때, 현재 틱과 `last_stall_start`의 차이를 계산하여 `mem_stall_ticks`에 누적합니다.

### Phase 3: 지표 계산 로직 (`kernel/trap.c`)
리눅스의 PSI처럼 주기적으로 `avg10`, `avg60` 등의 수치를 계산합니다.
*   **타이머 인터럽트 활용:** 매 타이머 틱마다 실행되는 `usertrap()` 또는 `kerneltrap()`에서 시스템 전체의 스톨 상태를 확인합니다.
*   **지수 평균법 적용:** 6주차에 학습한 SJF 스케줄링의 버스트 예측 공식인 $\tau(n+1) = \alpha \cdot t(n) + (1-\alpha) \cdot \tau(n)$을 사용하여 메모리 압박 지표를 갱신합니다.
    *   `some`: 최소 하나 이상의 프로세스가 메모리 대기 중인 시간 비율.
    *   `full`: 모든 실행 프로세스가 메모리 대기 중인 시간 비율.

### Phase 4: 사용자 모드 인터페이스 (System Call)
커널에서 계산된 PSI 데이터를 사용자 모드 앱(OOM 모니터 등)이 읽을 수 있도록 합니다.
*   **시스템 콜 추가:** `get_mem_pressure()` 시스템 콜을 구현하여 커널 내의 PSI 통계 구조체를 사용자 공간으로 복사합니다.

## 3. OOM 감지 및 대응 계획
*   **주기적 확인:** 커널 내부의 별도 백그라운드 태스크나 타이머 인터럽트 핸들러에서 PSI 지표가 임계치(Threshold)를 넘었는지 실시간으로 감시합니다.
*   **대응 로직:** `full_avg10` 수치가 15.0(15%) 이상으로 지속될 경우, 시스템 안정성을 위해 우선순위가 낮은 프로세스를 강제 종료(Kill)하거나 사용자에게 경고를 보냅니다.

## 4. 기대 효과
*   **정확성:** 사용자 모드 스레드에서 파일을 읽는 오버헤드 없이 커널이 직접 자원 상태를 파악하므로 반응 속도가 빠릅니다.
*   **안정성:** 메모리 고갈 전 단계(PSI 수치 상승)에서 미리 대응함으로써 시스템 전체가 멈추는 상황을 방지할 수 있습니다.
